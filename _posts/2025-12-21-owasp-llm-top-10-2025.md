## Executive Summary

OWASP(Open Worldwide Application Security Project)가 2025년 LLM 애플리케이션 Top 10 취약점 목록을 발표했습니다. 이번 업데이트는 급변하는 AI 위협 환경을 반영하여 4개의 새로운 취약점이 추가되었습니다.

## 2025년 신규 취약점 상세 분석

### 1. System Prompt Leakage (시스템 프롬프트 유출)

시스템 프롬프트가 노출되면 공격자가 LLM의 내부 동작 방식을 파악하여 더 정교한 공격을 수행할 수 있습니다.

**위험 시나리오:**
- 내부 지침 및 비밀 정보 노출
- 보안 가드레일 우회 방법 파악
- 비즈니스 로직 역공학

**대응 방안:**
- 시스템 프롬프트에 민감 정보 포함 금지
- 프롬프트 유출 탐지 메커니즘 구현
- 정기적인 레드팀 테스트

---

### 2. Vector and Embedding Weaknesses (벡터 및 임베딩 취약점)

RAG(Retrieval-Augmented Generation) 시스템의 급속한 확산으로 벡터 데이터베이스 보안이 중요해졌습니다.

**공격 유형:**
- **비인가 접근**: 벡터 DB에서 민감 데이터 추출
- **데이터 오염**: 악의적 임베딩 주입
- **행동 조작**: RAG 검색 결과 조작으로 모델 응답 변조

**대응 방안:**
- 벡터 DB 접근 제어 및 파티셔닝
- 임베딩 데이터 정기 감사
- 검색 결과 모니터링 및 로깅

---

### 3. Misinformation (허위정보)

LLM의 환각(Hallucination) 현상이 심각한 허위정보 확산으로 이어질 수 있습니다.

**주요 원인:**
- 학습 데이터의 편향
- 맥락 이해 부족으로 인한 사실 왜곡
- 통계적 추론에 의한 "그럴듯한" 거짓말 생성

**대응 방안:**
- 사실 확인(Fact-checking) 레이어 구현
- 출처 명시 기능 강화
- 사용자에게 AI 생성 콘텐츠 명확히 표시

---

### 4. Unbounded Consumption (무제한 리소스 소비)

기존 Model DoS를 대체한 더 포괄적인 개념으로, 리소스 관리 실패로 인한 비용 폭증과 서비스 중단을 다룹니다.

**공격 벡터:**
- 과도하게 큰 입력으로 시스템 과부하
- 무제한 API 호출로 비용 폭증
- 무한 루프 유도로 리소스 고갈

**대응 방안:**
- API 호출 속도 제한(Rate Limiting)
- 입력 크기 및 복잡도 검증
- 리소스 사용량 자동 모니터링 및 차단

## 2024 대비 주요 변화 분석

### 상승한 취약점

| 취약점 | 변화 | 이유 |
|--------|:----:|------|
| Sensitive Information Disclosure | #6→#2 | 기업 LLM 도입 확대로 데이터 유출 위험 급증 |
| Supply Chain | #5→#3 | 외부 API, 라이브러리, 데이터셋 의존성 증가 |

### 제거된 취약점

- **Model Denial of Service**: Unbounded Consumption으로 통합
- **Insecure Plugin Design**: 표준화된 보안 관행으로 우선순위 하락
- **Overreliance**: 광범위한 리스크에 흡수
- **Model Theft**: Sensitive Information Disclosure에 포함

## AICRA 권장사항

### 즉시 조치 사항

1. **프롬프트 인젝션 방어 강화**
   - 입력 검증 및 새니타이징
   - 시스템/사용자 프롬프트 분리
   
2. **민감 정보 유출 방지**
   - 학습 데이터 검토 및 정화
   - 출력 필터링 구현

3. **공급망 보안 점검**
   - 서드파티 의존성 감사
   - SBOM(Software Bill of Materials) 작성

### 중장기 로드맵

- RAG 시스템 보안 아키텍처 수립
- LLM 보안 모니터링 체계 구축
- 레드팀/블루팀 훈련 프로그램 도입
