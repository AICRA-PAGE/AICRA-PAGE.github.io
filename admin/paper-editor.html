1. 초록 (Abstract)
목적: 논문 전체를 150-300단어로 요약. 독자가 읽을지 결정하는 핵심.
필수 포함 (CGSRC):
- Context: 연구 배경 (1-2문장)
- Gap: 기존 연구의 한계 (1문장)
- Solution: 제안 방법 (1-2문장)
- Result: 핵심 수치 결과 (1문장, 숫자 필수)
- Contribution: 의의 (1문장)
금지: 인용, 수식, 그림 참조, "본 논문에서는"으로 시작
배경. 그러나 기존 연구는 [한계]. 본 연구는 [방법]을 제안하여 [지표]를 [수치]% 개선하였다. 이는 [의의]에 기여한다.
제안 기법은 BERT 기반 탐지기 대비 F1을 4.2%p 향상시켰다.
우리 방법이 기존보다 더 좋은 성능을 보였다.
2. 서론 (Introduction)
목적: 왜 이 연구가 필요한지, 무엇을 기여하는지 설득.
필수 구조:
- 1단락: 문제 영역 + 왜 중요한가 (실제 피해/비용 근거)
- 2단락: 기존 접근법의 한계 (구체적 gap)
- 3단락: 본 연구의 접근법 + 핵심 아이디어
- 4단락: 기여 목록 (C1, C2, C3)
- 5단락: 논문 구성 안내
금지: "중요하다"만 반복, 기여 없이 구조만 나열
기여. 본 논문의 주요 기여는 다음과 같다: - C1: [새로운 기법/프레임워크] - C2: [이론적/실증적 분석] - C3: [실험을 통한 검증]
"2024년 LLM 기반 피싱 공격으로 인한 피해액은 전년 대비 340% 증가하였다 [cite:1]."
"AI 보안은 매우 중요한 분야이다."
3. 관련 연구 (Related Work)
목적: 기존 연구를 체계적으로 분류하고, 본 연구와의 차별점을 명확히.
필수 포함:
- 분류 축 설정 (방법론/가정/데이터셋/위협모델)
- 각 범주별 대표 연구 비교
- 비교표 (Table) 포함 권장
- 기존 연구의 공통 한계 -> 본 연구의 차별점
금지: "A는 ~했다. B는 ~했다." 나열식 서술, 비교 없는 요약
"기존 방어 기법은 크게 필터링 기반 [cite:2-4]과 모델 수정 기반 [cite:5-7]으로 분류된다. 전자는 우회가 용이하고, 후자는 성능 저하가 크다는 공통 한계가 있다."
"Kim [cite:1]은 ~했다. Lee [cite:2]는 ~했다. Park [cite:3]은 ~했다."
4. 위협 모델 (Threat Model)
목적: 공격자의 능력과 가정을 명확히 정의하여 연구 범위를 한정.
필수 5요소:
- 자산 (Assets): 보호 대상
- 공격자 (Adversary): 목표, 접근 수준(black/gray/white-box), 능력
- 공격 표면 (Attack Surface): 진입점
- 가정 (Assumptions): 전제 조건
- 범위 외 (Out-of-scope): 고려하지 않는 것
금지: 가정 없이 "모든 공격 고려", 비현실적 공격자 모델
"공격자는 모델 API에 블랙박스 접근이 가능하며, 쿼리 예산은 1,000회로 제한된다."
"다양한 공격자를 모두 고려한다."
5. 방법론 (Methodology)
목적: 제안 기법을 재현 가능하게 기술.
필수 포함:
- 전체 시스템 개요도 (Figure)
- 입력-처리-출력 흐름 명시
- 핵심 수식 + 직관적 설명
- 알고리즘 의사코드 (재현 가능 수준)
- 하이퍼파라미터 + 선택 근거
금지: 코드 복붙, 구현 세부사항만 나열, 설계 근거 없는 선택
:::algorithm 제안 기법 Input: [입력 데이터] Output: [출력 결과] 1. [전처리 단계] 2. [핵심 처리 단계] 3. Return [결과] :::
"$\epsilon$=0.1로 설정한 이유는 예비 실험(Appendix A)에서 정확도-프라이버시 균형이 최적이었기 때문이다."
"$\epsilon$=0.1로 설정하였다." (근거 없음)
6. 실험 및 평가 (Evaluation)
목적: 제안 기법의 효과를 객관적으로 입증.
필수 순서:
- 실험 환경: HW/SW, 데이터셋, 전처리
- 평가 지표: 정의 + 선택 근거
- 비교 대상: 최신 baseline (2년 이내)
- 주요 결과: 표 + 수치 비교
- Ablation: 각 구성요소의 기여도
- 통계 검정: mean +/- std, 반복 횟수
금지: cherry-picking, 불공정 비교, 오차범위 미표시
"동일 GPU 예산(V100 24h) 하에서 5회 반복 실험한 결과, F1 92.3 +/- 0.8로 baseline 대비 4.2%p 향상."
"우리 방법이 전반적으로 우수하다." (수치 없음)
7. 논의 (Discussion)
목적: 결과의 의미, 한계, 영향을 분석.
필수 포함:
- 한계점 2개 이상 (솔직하게)
- 실패 사례 분석 (왜 실패했는지)
- 실제 배포 시 고려사항
- 오남용 가능성 + 방지 방안
금지: 한계 회피, "향후 연구로 남긴다"로만 넘김
"위협모델 가정이 깨질 경우(물리 접근) 탐지율이 62%로 하락하며, 이는 6.3절에서 정량 분석하였다."
"한계는 있으나 전반적으로 우수하다."
8. 결론 (Conclusion)
목적: 기여를 재확인하고 향후 방향 제시.
필수 포함:
- 연구 목적 재진술 (1문장)
- 핵심 기여 요약 (C1, C2, C3)
- 주요 수치 결과 재언급
- 향후 연구 방향 (구체적)
금지: 새로운 실험 결과, 서론 복붙, "지면 관계상 생략"
"향후 연구에서는 멀티모달 입력에 대한 방어를 확장하고, 실제 SOC 환경에서의 필드 테스트를 수행할 예정이다."
"향후 더 연구할 예정이다." (비구체적)

시제:
- 서론/배경: 현재 시제 ("~이다", "~한다")
- 방법론: 현재/과거 ("제안한다", "설계하였다")
- 실험결과: 과거 시제 ("향상되었다", "보였다")
- 결론: 현재 ("기여한다")
태: 능동태 우선, 수동태 남용 금지
종결어미: 한 편 내에서 통일 (합니다체 또는 이다체)

연구질문(RQ) / 가설(H) 설계
서론 끝에서 RQ/H를 번호로 명시하세요.
RQ1: [대상]에서 [요인]이 [결과]에 미치는 영향은? H1: [요인]이 증가하면 [결과]가 [방향]으로 변한다.
규칙: RQ는 탐색형, H는 검증형으로 구분. 각 가설은 측정 가능한 변수로 작성. 방법론/평가의 지표와 1:1 매핑.
실험설계 / 표본수 / 타당도
평가 섹션 필수 4블록:
1) 설계: 통제군/비교군, 독립/종속변수
2) 표본수 근거: 파워분석 또는 선행연구 기준
3) 무작위화/블라인드 여부
4) 타당도 위협: 내부/외부/구성/결론 타당도
표본수는 효과크기 d=..., 검정력 0.8, 유의수준 0.05 기준으로 산정하였다.
통계 결과 보고 표준
필수: 검정종류, 통계량, 자유도, p값, 효과크기, 신뢰구간
t(df)=값, p=값, d=값, 95% CI [하한, 상한] F(df1, df2)=값, p=값, eta^2=값
규칙: p<.001은 허용, 그 외 p=.023처럼 소수점 3자리. 효과크기 병기 필수. 유의하지 않은 결과도 보고.
통계 해석 (과장 금지)
원칙: 통계적 유의성 != 실질적 중요성
"유의한 차이가 관찰되었다. 다만 효과크기는 중간 수준이다."
"완벽히 증명되었다", "혁신적으로 개선되었다"
